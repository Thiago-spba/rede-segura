## 📌 Contexto Acadêmico

> Este projeto foi desenvolvido como **trabalho de conclusão da Fase 03** da disciplina de **Redes de Computadores**, no curso de **Engenharia de Computação** do Centro Universitário Celso Lisboa.

A proposta consistia em criar uma **consultoria fictícia de segurança de redes** — aqui representada pela empresa **Master Security** — e elaborar uma análise técnica comparando três abordagens de proteção de tráfego de dados na internet.

---

## 🎯 Objetivo do Projeto

Mapear e comparar três arquiteturas de rede sob os aspectos de **privacidade**, **criptografia** e **rastreabilidade**, orientando a tomada de decisão sobre a melhor solução de segurança para uma empresa.

---

## 🔍 O que foi analisado

| # | Tecnologia | O que faz |
|---|---|---|
| 01 | **Fluxo Regular** | Conexão direta sem proteção — IP exposto, ISP rastreia tudo |
| 02 | **Servidor Proxy** | Intermediário que oculta o IP, mas sem criptografia local |
| 03 | **VPN** | Túnel criptografado ponta a ponta — proteção total |

---

## ✅ Resultado da Análise

```
Fluxo Regular  →  Proteção Baixa   ● ○ ○
Proxy          →  Proteção Média   ● ● ○
VPN            →  Proteção Alta    ● ● ●  ← Recomendado
```

A **VPN** foi apontada como a solução mais completa: oculta o IP do usuário, impede o rastreamento pelo provedor (ISP) e garante criptografia AES-256 em todos os dados transmitidos.
